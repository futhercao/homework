🌱 对环境与社会可持续发展的影响

本系统在工程实现与应用价值两个层面践行可持续发展理念:一方面通过异步爬取、礼貌抓取、 去重与缓存复用、CPU 友好模型降低计算与带宽能耗(环境可持续);另一方面,信息检索与 灾害信息抽取本身服务于应急响应与防灾减灾(社会可持续)。

♻️ 环境可持续 — 降低能耗与资源占用

异步并发aiohttp 异步爬虫 + 并发上限,单位时间内用更少进程/线程完成抓取,降低 CPU 空转与能耗。
礼貌抓取UA 轮换 + 随机请求间隔 + 单域名限流,减轻目标站点负载,避免无效重试带来的带宽浪费。
去重SimHash 近似去重 + URL 去重,避免重复抓取与重复入库,直接减少网络请求与存储。
断点续爬逐篇落盘、同 URL 覆盖同名文件,中断可续、重跑不产生冗余下载。
缓存复用倒排索引、CLIP 向量索引、OCR 结果均持久化到本地,演示与重跑直接复用,避免重复计算(OCR/编码是主要算力开销)。
CPU 友好全流程纯 CPU 可跑:BM25/VSM 轻量检索、ViT-B/16 量级中文 CLIP、EasyOCR;无需 GPU,降低硬件门槛与功耗。

🤝 社会可持续 — 服务应急响应与防灾减灾

信息检索面向海量真实新闻的高效检索(BM25/语义/混合),帮助公众与决策者快速获取权威信息,提升信息可达性与社会知情权。
灾害抽取自动从灾害报道中结构化"时间/地点/伤亡/损失/响应/救援"等要素,可辅助应急部门快速掌握灾情、统计与复盘。
知识图谱将分散报道关联为事件图谱,支持按地区/类型/时间的态势分析,服务防灾减灾与历史灾情研究。
多媒体抽取OCR 识别灾情图片中的文字(响应级别、统计表、通告),补全纯文本遗漏的信息,提升应急信息完整度。

📐 可量化的"省"

  • 向量/索引一次构建、持久化复用:CLIP 编码 1500+ 图、语义编码千级文档仅需一次性离线计算,在线查询免重复编码。
  • OCR 结果缓存:同一图片只识别一次,后续演示零额外算力。
  • 去重过滤:爬取阶段即剔除重复/非相关页面,减少下游解析、索引、抽取的无效计算。

注:以上为系统设计层面的可持续性措施,旨在以更低的算力/带宽/能耗达成同等效果,并使技术服务于公共安全。
