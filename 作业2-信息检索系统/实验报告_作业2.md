# 信息检索系统 实验报告(作业2)

## 摘要

本实验构建了一个面向**真实中文新闻语料**的信息检索系统,实现了 **BM25、向量空间模型(VSM)、混合检索(Hybrid)** 三种检索算法,
引入 **伪相关反馈(PRF)查询扩展**,并扩展到 **中文 CLIP(Chinese-CLIP)跨媒体检索**(中文以文搜图 / 以图搜文)。
系统配备**人工相关性评价**界面(逐结果打分,实时计算 P@k / MAP / nDCG)与**自动基准评价**脚本。
文档语料由自研**异步爬虫**抓取,共 **802 篇**真实新闻(来自**新华网 / 人民网 / 央视网 / 中国新闻网 / 光明网 / 中国经济网 / 科技日报 / 澎湃**等权威媒体,覆盖 **27 个子域名**),**100% 含真实 URL 与日期、可逐条溯源**。以文搜图图库在新闻配图基础上**另补充主题图以扩充召回**,经 **VLM 视觉大模型质检**后 **519 张**入 CLIP 索引;查询时再由 **Qwen-VL 对候选图逐张做相关性精排**,使"以文搜图"只返回真正相关的图片。
**核心检索(BM25 / VSM / 混合 / 评价)全流程纯 CPU 可运行**;VLM 质检与精排为可选增强(结果带缓存,无密钥时优雅降级)。

## 二、爬虫设计(数据采集)

爬虫是检索系统的数据基石,本系统采用**异步高并发 + 礼貌抓取 + 多级去重**的工程化设计:

1. **异步并发**:基于 `aiohttp` 协程并发抓取,并发上限可配,单位时间吞吐高、CPU 空转低。
2. **多源种子 + 链接发现**:从多家主流门户/垂直站点的多个频道(财经、社会、健康、教育、科技等)出发,BFS 发现正文链接,保证**来源与主题多样性**。
3. **正文抽取**:`readability-lxml` 提取新闻正文,剥离导航/广告/评论噪声。
4. **近似去重**:对正文计算 **SimHash** 指纹,海明距离阈值内判为重复;URL 级去重防重复入库。
5. **礼貌抓取**:User-Agent 轮换 + 随机请求间隔 + 单域名限流,降低目标站点压力。
6. **断点续爬**:逐篇落盘,同 URL 覆盖同名文件,中断可续、重跑不产生冗余下载。
7. **配图下载**:解析正文 `<img>`,按 PNG/JPG 文件头magic校验后落盘,为多模态检索提供素材。

## 三、核心检索算法

### 3.1 BM25(Okapi)

对查询 $Q=\{q_1,...,q_m\}$ 与文档 $D$:

$$\text{score}(D,Q)=\sum_{i=1}^{m} \text{IDF}(q_i)\cdot\frac{f(q_i,D)\cdot(k_1+1)}{f(q_i,D)+k_1\cdot\left(1-b+b\cdot\frac{|D|}{\text{avgdl}}\right)}$$

其中 $\text{IDF}(q_i)=\ln\frac{N-n(q_i)+0.5}{n(q_i)+0.5}+1$,参数 $k_1=1.5,\ b=0.75$。BM25 同时建模**词频饱和**与**文档长度归一化**。

### 3.2 向量空间模型(VSM)

文档/查询表示为 **TF-IDF** 加权向量,相关性用**余弦相似度**度量:

$$\text{sim}(D,Q)=\frac{\vec{D}\cdot\vec{Q}}{\|\vec{D}\|\,\|\vec{Q}\|}$$

### 3.3 混合检索(Hybrid, RRF 融合)

并行执行 **BM25 + VSM + 语义召回**(`sentence-transformers` 多语言向量),用 **倒数排名融合(Reciprocal Rank Fusion)** 合并:

$$\text{RRF}(d)=\sum_{r\in\{bm25,vsm,sem\}}\frac{1}{k+\text{rank}_r(d)},\quad k=60$$

三路召回互补:BM25 重词项精确匹配,VSM 重向量相似,语义召回缓解**词汇鸿沟**(同义/近义)。

### 3.4 伪相关反馈(PRF)查询扩展

将初检 Top 文档视为伪相关集,统计候选词的 $tf\cdot idf$ 权重(剔除原查询词、长度过短词),
取权重最高的若干词扩展查询,**提升召回**。页面提供"查询扩展"开关,扩展词回显。

## 四、实验结果

### 4.1 自动基准评价(BM25 vs VSM)

用 10 个跨领域查询(人工智能、地震救援、新能源汽车、芯片制造、台风灾害、高等教育、医疗健康、自动驾驶、自然灾害损失、财经投资),
采用**与 BM25 解耦的启发式相关性判定**(正向关键词命中 ≥3 部分相关 / ≥5 高相关,负向关键词跨域降级,标题命中加成,避免循环验证),10 查询平均:

| 指标 | BM25 | VSM | 优胜 |
|------|------|-----|------|
| P@5 | **0.5000** | 0.2800 | BM25 |
| P@10 | **0.3800** | 0.2100 | BM25 |
| F1@10 | **0.4529** | 0.2891 | BM25 |
| MAP | **0.5518** | 0.4206 | BM25 |
| nDCG@10 | **0.5436** | 0.4307 | BM25 |
| MRR | **0.7667** | 0.5202 | BM25 |

**结论**:BM25 在全部指标上优于纯 VSM,其词频饱和与长度归一化对新闻检索更稳健;混合检索在此基础上再融合语义召回,
缓解词汇鸿沟。该基准可通过 `python scripts/run_evaluation.py` 复现。

### 4.2 人工评价(网页按钮)

检索结果页每条结果旁设 **相关 / 部分相关 / 不相关** 按钮,评分者点击后**实时计算** P@5、P@10、MAP、nDCG@10、MRR 并入库(`/evaluate` 汇总)。
此为面向真实使用者的主观相关性评价,与 4.1 的自动基准互为补充。

## 五、多模态跨媒体检索(创新/多媒体)

采用中文原生 **Chinese-CLIP**(经 `transformers` 加载 `OFA-Sys/chinese-clip-vit-base-patch16`,**ViT-B/16 视觉塔 + 中文 RoBERTa 文本塔**),将图像与**中文文本**映射到同一 512 维联合空间:

- **以文搜图**:中文描述经中文文本塔编码后,在 519 张内容图向量(FAISS 余弦)中召回候选,再经 VLM 相关性精排(见 5.3);
- **以图搜文**:上传图片编码后,与文档关联图像比对,召回相关文档。

> **本次优化**:由多语言通用 CLIP(`xlm-roberta-base-ViT-B-32`)升级为**中文原生 Chinese-CLIP**。后者在 ~2 亿中文图文对上训练,文本塔为中文 RoBERTa,中文查询与图像的语义对齐显著优于"多语言文本塔 + 英文视觉预训练"的折中方案;模型体积更小(~600MB vs ~1GB),更适合纯 CPU 部署。系统通过 `clip_backend` 配置项保留多语言 CLIP 作回退,便于对照。

### 5.1 图片质检:VLM 视觉大模型预处理(创新点)

新闻页抓取的配图中混有大量"看着是图、实为噪声"的内容——栏目快讯图(纯文字"最新消息/快讯")、广告横幅、网站 logo、二维码、网页/App 界面截图、纯装饰素材等。仅靠"来源 URL 关键词 + 最小边像素"的廉价规则无法识别它们,导致**以文搜图结果被大量无关图污染**。

为此引入一道**语义级质检**:将每张图(降采样至 512px)送入**视觉大模型**(通义千问 `qwen3-vl-plus` 视觉模型,经 DashScope OpenAI 兼容接口调用),按统一提示词判定其是否为"有意义的新闻内容图",仅输出结构化 JSON `{keep, type, reason}`,只保留通过者再进入 CLIP 索引。工程上:

- **并发 + 缓存**:线程池并发判定(实测 8 并发约 1 张/秒),结果按文件名持久化到 `data/vectors/vlm_verdicts.json`,重复构建零调用、随交付分发,**无密钥也能复现"已清洗"的索引**;
- **稳健降级**:多次失败则 fail-open(默认保留)且不写缓存,单次网络抖动不会误删图;密钥仅取环境变量 `DASHSCOPE_API_KEY`,**不写入任何文件**;
- **效果**:全库 **846 张**候选图先经廉价过滤(URL 垃圾关键词 / 最小边 <200px)剔除 **149 张** → 697 张,再送 VLM 质检剔除 **178 张** → 最终 **519 张**内容图入 CLIP 索引。被 VLM 剔除的主要类别为:与新闻无关通用素材图、纯装饰图(纯色/底图)、纯文字图/海报、网页/App 界面截图、站标 logo/品牌标识/图标、二维码/水印、AI 生成图、广告横幅等。清洗后以文搜图的结果相关性明显提升(典型查询 Top 结果不再出现纯文字海报/广告图)。

> **质检结论复用(全站一致过滤)**:同一份 `vlm_verdicts.json` 不仅用于 CLIP 索引,还**复用于文档详情页配图展示与检索结果"🖼N张图片"角标计数**——按内容图口径(廉价过滤 + VLM 质检)过滤后,**519 张**真实内容图入索引,其余 logo/广告/海报/截图等一律隐藏,使详情页与结果角标也不再混入无关图。采用**黑名单(逐图 fail-open)语义**:仅剔除被明确判否者,verdicts 缺失或未覆盖时自动回退为保留,换语料、可移植均稳健。

FAISS 索引采用 `serialize_index` 字节序列化持久化,规避 Windows 中文路径下 `write_index` 失败的问题。

### 5.2 模态gap 与"阈值失灵"问题

CLIP 跨模态检索本质是"返回与查询向量**最相近**的 top-k 图片"。但中文 CLIP 存在**模态gap**:图、文向量分布在
联合空间的不同区域,使图文余弦被压缩在很窄的区间(本库实测约 **0.33–0.48**)。其后果是**纯余弦阈值会同时漏判与误纳**:
实测无关图(机械零件、金字塔示意、纯文字标识)的余弦可达 **0.377–0.385**,反而**高于**部分真实弱相关图——这正是
"搜地震却混入无关图"的根因。因此**任何固定余弦阈值都无法干净地分开相关/无关**,这是模型层面的固有现象,调阈值治标不治本。

### 5.3 查询级 VLM 相关性精排(根治,创新点)

为根治上述问题,采用"**召回—精排**"两段式:CLIP 只负责**召回**(快,把可能相关的图都捞上来),由视觉大模型负责
**精排**(准,逐张裁定是否真相关)。

- **召回**:CLIP 以一个宽松的"召回下限"(`vlm_judge_recall_floor`,默认 **0.28**,仅用于滤掉明显噪声、省 API 调用)取 Top-N 候选;
- **精排**:对每张候选图,将图片(降采样 512px)连同查询词送入 **Qwen-VL(`qwen3-vl-plus`)**,令其站在"用此词搜图的用户"
  视角判定是否相关,输出结构化 JSON `{match, conf, desc}`;仅保留 `match=true 且 conf≥0.55` 者并按置信度重排。判定按
  **(查询词, 文件名)** 缓存到 `data/vectors/vlm_judge_cache.json`,同词重复搜索零 API 调用(演示秒回);无密钥时优雅降级
  为纯余弦阈值,保留可对照的旧行为。前端逐图显示 AI 判定(如"✅ AI 判定相关 95%「四川芦山地震震中示意图」")并统计
  "候选 N → 相关 M、剔除 K",**让"AI 对检索结果的评价"可见、可解释**。

**实测(默认精排开启)**:

| 查询 | CLIP 召回 | AI 判定相关 | 剔除无关 | 典型保留(AI 描述) |
|------|----------|------------|---------|------------------|
| 地震 | 12 | **12** | 0 | 芦山 7.0 级震中示意图、倒塌废墟救援搜救 |
| 洪水 | 12 | **12** | 0 | 洪水淹没街道乘舟救援、车辆涉水 |
| 台风 | 12 | **12** | 0 | 巨浪拍岸、强风中树木弯折、卫星云图 |
| 火灾 | 12 | **12** | 0 | 熊熊烈火、消防员扑救、山林野火 |
| 新能源汽车 / 芯片 | 12 | **12** | 0 | 充电站车辆、晶圆芯片特写 |
| **篮球 / 咖啡**(库中无此题材) | 12 | **0** | 12 | (全部判为无关 → 如实返回空,不堆无关图) |

与 5.1 的入库质检互补:5.1 离线按"是否内容图"清洗整库,5.3 在线按"是否与本次查询相关"精排——前者保证库干净,后者保证
**每次结果**只含真正相关图。这一步把"以文搜图"的精度从"靠阈值碰运气"提升为"AI 逐张把关"。

## 六、对环境与社会可持续发展的影响

- **环境可持续**:异步并发 + 礼貌抓取 + SimHash 去重 + 索引/向量**一次构建持久化复用** + **纯 CPU 友好模型**,以更低算力/带宽/能耗达成同等检索效果。
- **社会可持续**:高效检索提升海量真实信息的**可达性**与公众知情权;多模态检索降低信息获取门槛。

详见系统 `/sustainability` 页面。

## 七、局限与展望

- 语料来源已升级为权威媒体(新华网/人民网/央视网/中新网/光明网/经济网/科技日报等),但人民网(~27%)与中国新闻网(~22%)占比偏高,后续可加权采样均衡来源;灾害题材现仅 6 篇,可针对应急管理/气象频道定向补采;
- 自动基准的相关性判定为启发式,可引入更多人工标注构建标准测试集;
- 已采用中文原生 Chinese-CLIP + 查询级 Qwen-VL 精排,基本消除"以文搜图出无关图";局限在于精排依赖在线 VLM(已用 (查询词,文件名) 缓存与无密钥降级缓解),图库题材覆盖仍可继续扩充;垂直领域(如灾害图像)亦可对 CLIP 继续微调,以减少送审候选、进一步降低时延与调用量。
