# 验收指南 · 作业2 信息检索系统

> 本系统独立运行，端口 **8090**。若同时验收作业2 与作业3，请**分别启动、一次只跑一个**（同端口）。
> 带中文的查询建议**在浏览器里输入**（避免 Windows 终端编码问题）；纯英文 / 文档ID 的接口可直接用 `curl` 验证。

## 0. 前置与启动

```bash
pip install -r requirements.txt     # 纯 CPU，无需 GPU
python run.py web                    # 仓库已附带构建好的索引与数据
# 浏览器打开 http://127.0.0.1:8090
```

> 离线说明：倒排索引 / BM25 / VSM / 混合检索 / 查询扩展 / **人工评价**均为纯本地计算，**无需联网**。
> 仅 `/multimodal`（中文 CLIP）首次需联网下载模型（~600MB）到本地缓存；无网络时可只验收文本检索部分，不影响基本要求与人工评价档。

## 0.5 现场演示：爬虫真实可爬（对应「爬取数据」评分项 · 需现场解释演示）

> 评分表「爬取数据」一栏需现场演示**爬虫确实能爬**，并口头解释其设计。本系统语料（802 篇）即由下方爬虫真实抓取（**中文** + **命令行**）。

**A. 当场跑给老师看（推荐用安全演示脚本，不污染已交付语料）：**

```bash
python scripts/crawl_demo.py            # 实时抓取 5 篇真实新闻并打印 来源/标题/日期/正文长度/配图数
python scripts/crawl_demo.py --max 8    # 自定义篇数
```

- 该脚本把抓取结果写入**系统临时隔离目录**（用完即弃），**不改动** `data/documents/` 的 802 篇与已建索引，因此演示后语料规模与报告数字一致。
- 约 30–90 秒返回，可见**当天日期**的真实文章（如人民网 / 中新网 / 科技日报），证明是实时爬取而非读缓存。
- 完整全量爬取用 `python run.py crawl --max 800`（会写入交付语料目录，仅在需要重建语料时使用）。

**B. 现场解释要点（爬虫设计 / 评分官会问）：**

| 设计点 | 实现位置 | 一句话解释 |
|---|---|---|
| 异步高并发 | `src/crawler/engine.py` | `aiohttp` + `asyncio.Semaphore` 控并发，单域名限流 |
| 反反爬 | `engine._fetch` / `config.py` | 15 个 User-Agent 轮换 + Referer 伪装 + 失败重试 / 指数退避 |
| 多源种子 | `config.py: seed_urls` | 新华网/人民网/央视网/中新网/光明网/中国经济网/科技日报 等 **权威采编源**（非单一站点） |
| 两阶段抓取 | `engine.crawl` | Phase 1 频道页收集候选链接 → Phase 2 并发抓正文 + 浅层 BFS 发现新链接 |
| 去重 | `src/crawler/dedup.py` | SimHash 近似去重，避免转载重复入库 |
| 真实多媒体 | `engine._download_images` | 顺带下载文章配图到本地，供以文搜图 / 以图搜文 |

## 2. 附：命令行快速自检（可选，ASCII 安全）

```bash
# 语料规模（基本要求 ≥100 篇）
curl -s http://127.0.0.1:8090/api/stats

# 检索返回的五个必列字段（score/title/snippet/url/date）
curl -s "http://127.0.0.1:8090/api/search?q=ai&algo=bm25" | head -c 600

# 人工评价指标非 0（81–90 档；本次修复点）——传入排序结果即可算出 P@k/MAP
curl -s -X POST http://127.0.0.1:8090/api/evaluate/search \
  -H "Content-Type: application/json" \
  -d '{"query":"ai","algorithm":"bm25","results":[{"doc_id":"A"},{"doc_id":"B"},{"doc_id":"C"}],"judgments":{"A":2,"B":1}}'
# 预期：metrics 中 P@5、MAP、MRR 等 > 0

# 多媒体以文搜图（需已下载 CLIP 权重）
curl -s "http://127.0.0.1:8090/api/multimodal/search-images?q=flood&top_k=5" | head -c 400
```

> 注：第 7 条更直观的验收方式是直接在网页上点评价按钮看指标；curl 仅用于无人值守的快速回归。
