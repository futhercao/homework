# 验收指南 · 作业3 灾害信息抽取系统

> 本系统独立运行，端口 **8090**。若同时验收作业2 与作业3，请**分别启动、一次只跑一个**（同端口）。
> 纯英文 / 文档ID 的接口可直接用 `curl` 验证；页面操作在浏览器里完成。

## 0. 前置与启动

```bash
pip install -r requirements.txt     # 纯 CPU，无需 GPU
python run.py web                    # 仓库已附带抽取结果 / 知识图谱 / OCR 缓存
# 浏览器打开 http://127.0.0.1:8090
```

> 离线说明：抽取 / 事件组装 / 知识图谱 / **逐字段人工评价**均读取仓库已附带的预计算结果，**无需联网即可完整演示**。
> 仅「OCR 实时重新识别」按钮与从零 `run.py ocr/extract` 需要模型，首次运行会联网下载 EasyOCR 到本地缓存（NER 用 jieba 词性标注，无需下载）；无网络时直接用预置 OCR 结果演示即可。

## 0.5 现场演示：爬虫真实可爬（对应「爬取数据」评分项 · 需现场解释演示）

> 评分表「爬取数据」一栏需现场演示**爬虫确实能爬**，并口头解释其设计。本系统灾害语料（118 篇真实灾害事件）即由下方爬虫真实抓取（**中文** + **命令行**）。

**A. 当场跑给老师看（推荐用安全演示脚本，不污染已交付语料）：**

```bash
python scripts/crawl_demo.py            # 实时抓取 5 篇真实灾害新闻并打印 来源/灾害类型/日期/正文长度/配图数
python scripts/crawl_demo.py --max 8    # 自定义篇数
```

- 该脚本把抓取结果写入**系统临时隔离目录**（用完即弃），**不改动** `data/disaster_docs/` 的 118 篇、`extractions.json` 与知识图谱，因此演示后语料规模与报告数字一致。
- 约 1–2 分钟返回，可见**近日日期**的真实灾害报道（如应急管理部 / 新华网 / 人民网），证明是实时爬取而非读缓存。
- 完整全量爬取用 `python run.py crawl --max 200`（会写入交付语料目录，仅在需要重建语料时使用）。

**B. 现场解释要点（灾害爬虫设计 / 评分官会问）：**

| 设计点 | 实现位置 | 一句话解释 |
|---|---|---|
| 多入口召回 | `scripts/crawl_disaster.py: DISASTER_QUERIES / SEED_URLS` | 百度新闻搜索（13 个灾害关键词）+ 应急部/消防局/水利部/气象局 等权威频道种子 |
| 权威源白名单 | `crawl_disaster.py: SOURCE_WHITELIST` | 仅保存 gov.cn/新华/人民/央视/中新/光明/央广/澎湃，滤掉门户与自媒体 |
| 三重过滤 | `is_disaster_article` → `RegexExtractor` → `is_disaster_event` | 关键词初筛 → 正则校验真实灾害类型 → 事件性过滤（剔除党务/会议/科普/导航） |
| 浅层 BFS | `extract_news_links` | 在已确认灾害页面顺带发现更多文章链接，提升数量与域名多样性 |
| 单源配额 | `SOURCE_CAP=35` | 单一来源最多 35 篇，强制来源/题材多样性 |
| 真实多媒体 | `download_images` | 按文件魔数判定下载文章真实配图，供 EasyOCR 抽取 |

## 2. 附：命令行快速自检（可选，ASCII 安全）

```bash
# 语料规模（基本要求 ≥100 篇）+ 有效事件 / OCR 参与数
curl -s http://127.0.0.1:8090/api/stats

# 单篇抽取：7 要素组成事件 + 至少正则（用真实文档ID）
curl -s -X POST http://127.0.0.1:8090/api/extract -d "doc_id=dis_0ef962e3bc6e" | head -c 800

# 逐字段人工评价（81–90 档）
curl -s -X POST http://127.0.0.1:8090/api/evaluate/extraction \
  -H "Content-Type: application/json" \
  -d '{"doc_id":"dis_0ef962e3bc6e","predictions":{"disaster_type":"earthquake"},"judgments":{"disaster_type":"correct","event_time":"partial"}}'
# 预期：返回 metrics.accuracy 及 correct/partial/wrong 计数

# 知识图谱规模（91–100 档创新）
curl -s http://127.0.0.1:8090/api/kg | head -c 200
```

> 注：第 6、8 条更直观的验收方式是直接在网页上操作；curl 仅用于无人值守的快速回归。
> 文档ID 可在首页语料列表中任取，或用上面的 `dis_0ef962e3bc6e`。
