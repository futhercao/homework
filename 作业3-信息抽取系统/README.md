# 灾害信息抽取系统(作业3)

从**真实灾害新闻**中抽取结构化事件信息的系统:**集成抽取**(正则 + 触发词 + NER 加权投票)7 个灾情要素、
构建**知识图谱**、**EasyOCR 多媒体抽取**(识别配图文字并回灌进抽取)、以及**逐字段人工评价**。
全流程**纯 CPU 可跑,无需 GPU**。

> 灾害语料全部为爬虫真实抓取,**无任何编造/生成数据**。

> 📋 **验收 / 复现**:见 [`验收指南.md`](验收指南.md) —— 每一步操作对应一个作业需求与一个评分档位。

## 一、功能与评分点对应

| 模块 | 说明 | 入口 |
|------|------|------|
| 📋 集成抽取 | 抽取 **7 要素**(类型/时间/地点/伤亡/经济损失/响应级别/救援);**创新点**:正则 + 触发词 + NER **加权投票**,每字段给出集成置信度与来源 | `/` |
| 🕸 知识图谱 | 事件 ↔ 地点 ↔ 类型 ↔ 机构 力导向图谱(ECharts),支持按类型/地区态势分析 | `/kg` |
| 🔤 多媒体抽取 | **VLM 图片质检**（Qwen-VL 剔除 logo / 二维码 / UI 等无关图，91→20 张内容图）+ **EasyOCR**（ch_sim+en, CPU）识别保留内容图文字并**回灌进抽取**；支持"实时重新识别"现场验证 | `/ocr` |
| ✅ 人工评价 | 每个字段旁**打分按钮**(✅正确 / ⚠️部分 / ❌错误)→ 逐字段准确率统计 | `/evaluate-extraction` |
| 🌱 可持续发展 | 系统对环境(低能耗)与社会(防灾减灾)可持续的贡献 | `/sustainability` |

### 创新点:集成抽取(Ensemble)

单一正则容易漏抽、错抽。本系统融合三种互补的抽取器,按字段加权投票:

- **正则引擎**:模式精确,负责"响应级别""伤亡数字"等强格式字段;
- **触发词引擎**:基于触发词 + 上下文窗口,补正则覆盖不到的表述;
- **NER 引擎**:`jieba.posseg` 词性标注识别人名/地名/机构/时间 + 领域规则补全,补地点与机构字段(若装有百度 LAC 则自动改用 LAC 增强)。

投票时按字段权重合并候选(归一化/包含关系去重,取最高权重),并输出**置信度 + 来源标注**,
实测相比纯正则**信息完整度提升 ~4.8%**(平均 3.70 → 3.88 字段/篇)。

## 二、技术架构

```
灾害爬虫 (异步) ──→ 灾害语料 (data/disaster_docs) + 配图 (data/images)
                          │
            ┌─────────────┼──────────────┐
        OCR (EasyOCR)   集成抽取        知识图谱
        图片文字回灌 →  正则+触发词+NER → 事件 → 实体/关系图
                          │
                     7 要素结构化事件 + 逐字段人工评价
```

- **爬虫**:`aiohttp` 异步并发 + UA 轮换 + 单域限流 + `readability` 正文抽取 + **SimHash 去重** + 配图下载(按文件头校验)+ 双重灾害过滤。
- **抽取**:`EnsembleExtractor` 加权投票;命中预计算结果(`extractions.json`)秒级返回。
- **OCR**:EasyOCR 主引擎(纯 CPU),识别结果缓存到 `ocr_cache.json`;图片一律按文件名在本地 `data/images` 解析(可移植、规避 Windows Unicode 路径问题)。
- **知识图谱**:事件聚合为实体(事件/地点/类型/机构)与关系,持久化到 `knowledge_graph.json`。

## 三、安装与运行

```bash
# 1. 安装依赖(纯 CPU;easyocr 会自动拉取 torch/opencv)
pip install -r requirements.txt

# 2. 直接启动(仓库已附带抽取结果、知识图谱、OCR 缓存)
python run.py web
# 浏览器打开 http://127.0.0.1:8090

# (可选) 从零构建:
python run.py crawl --max 400   # 异步爬取真实灾害新闻
python run.py ocr               # EasyOCR 识别配图文字, 回灌
python run.py extract           # 全量集成抽取 + 构建知识图谱
```

> **离线/可移植性说明**:抽取、事件组装、知识图谱、**逐字段人工评价**均读取仓库已附带的预计算结果(`extractions.json` / `knowledge_graph.json` / `ocr_cache.json`),**无需联网即可完整演示**。
> 仅"OCR 实时重新识别"按钮与从零 `python run.py ocr/extract` 需要模型:**首次运行会从网络下载** EasyOCR(`ch_sim+en`)识别模型到本地缓存,之后离线复用(NER 用 jieba 词性标注,无需联网下载)。若批改环境无网络,直接用预置 OCR/抽取结果演示即可(不影响基本要求与人工评价档)。

## 五、数据真实性

- `data/disaster_docs/` 内每篇文档均带原始 `url`、`source`、`date`,可逐条溯源至真实新闻页面。
- `data/images/` 为爬虫从新闻页**实际下载**的灾情配图。
- 图片路径已规范化为相对路径(`data/images/<文件名>`),不含任何机器绝对路径,**可整体迁移**。
- 项目**不含任何数据生成器**,不存在编造文档。
